AI IDEツール利用効果分析レポート

1. 調査の背景

本レポートは、2025年9月末に当社開発者46名を対象に実施したアンケート調査を基に作成されました。回答者の83%（39名）が5年以上の経験を持つエンジニアであり、主にバックエンドおよびフロントエンド開発に従事しています。

2. 主な調査結果の要約

データ分析の結果、Gemini Code AssistはDr.JOY開発チームの期待および業務要件を満たしていないことが明らかになりました。

回答者の80%以上が「普通」または「悪い」と評価しており、その主な理由は「応答速度の遅さ」「コード提案の信頼性の低さ」「ユーザーエクスペリエンスの悪さ」に集約されます。

一方、Cursor AI Editorは、93%のユーザーがGemini Code Assistより遥かに優れていると回答しています。

提言：来年3月のGemini Code Assistの契約更新を見送り、リソースをCursor AIなど、より効果的なツールの試験導入に振り分けることを検討すべきです。

B. 核心的な問題点

  • 問題点1：応答速度が非常に遅い (最大の障害)

    これは最も多く寄せられた不満点です。

    「応答速度が遅い」
    「長時間ロードした後、応答がなくストレスを感じる」
    といった意見が繰り返し見られました。

  • 問題点2：提案の品質が低く、信頼性に欠ける

    多くの開発者が、提案されるコードのロジックが誤っている、または文脈に合っていないと報告しています。大規模なコードベースへの理解が浅く、実用的でない、あるいは誤った提案を生成する傾向があります。

    「提案が不正確／ロジックが間違っている」
    「リポジトリ全体や複数ファイルにまたがる作業能力に制限がある」
  • 問題点3：統合性の低さと劣悪なUX（ユーザーエクスペリエンス）

    多くのユーザーが、既存の開発ワークフローへの統合に困難を感じています。また、

    「テキストの表示が途切れる問題」
    「回答が長文で、操作性が悪い」
    といったUI上の問題も、利用体験を損なう要因となっています。

Cursor AIの強み：

  • 高精度かつ文脈に即した提案：コードベースへの深い理解力。
  • 高速な応答速度：開発フローを中断させない。
  • AIモデルの柔軟な選択：GPT-4やClaude Sonnet等をタスクに応じて切り替え可能。
  • スムーズなワークフロー統合：複数ファイルの扱いやチェックポイント機能など、高度な機能を搭載。
